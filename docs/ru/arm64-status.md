### Что добавляет PR

**OSTD (`ostd/src/arch/aarch64/`) :**
- `boot/` — точка входа BSP, таблицы страниц загрузки
- `mm/` — таблицы страниц ARM64 (4-уровневая paging), настройка MMU
- `task/` — переключение контекста, сохранение/восстановление FPU/SIMD
- `irq/` — контроллер прерываний GICv3 (использует сторонний crate)
- `timer/` — универсальный таймер ARM (физический EL1)
- `trap/` — обработка исключений EL1 (sync/IRQ/FIQ/SError)
- `cpu/` — функции CPU, SMP через PSCI
- `iommu/` — заглушка IOMMU
- `device/` — обнаружение устройств через FDT
- `io/` — абстракция MMIO
- `power.rs` — управление питанием PSCI (выключение/перезагрузка)
- `serial.rs` — консоль UART PL011

**Ядро (`kernel/src/arch/aarch64/`) :**
- Поддержка процессов / потоков
- Таблица системных вызовов (EL0 → EL1)
- Обработка TLS (TPIDR_EL0)
- Перечисление PCI
- Поддержка VirtIO
- Исправление бага сброса TLB в `KVirtArea::drop()`

**OSDK :**
- Сырой формат `Image` ARM64 для протокола загрузки Linux QEMU
- Схема QEMU Arm64 в `OSDK.toml`
- CI GitHub Actions для линта + компиляции arm64

## Стратегия kei

kei сливает эту ветку через git (не через патчи). Это означает, что :

1. Полное дерево `ostd/src/arch/aarch64/` существует в репозитории kei
2. Мы можем изменять любой файл напрямую
3. Синхронизация с апстримом — это `git merge`, а не `quilt push`
4. Когда апстрим в конечном итоге сольёт другую реализацию arm64, мы
   ребейзим наш BSP поверх нового архитектурного кода

## Известные проблемы в ветке arm64-support

| Проблема | Серьёзность | Действие kei |
|----------|-------------|--------------|
| Весь код сгенерирован LLM | Высокая | Аудит M2: проверить каждый файл, исправить артефакты |
| Сторонний crate GICv3 | Средняя | Заменить внутренним драйвером |
| Тесты только в QEMU | Высокая | Загрузка на реальном железе NanoPi R3S |
| Нет SMP/многоядерности | Средняя | Добавить запуск вторичного CPU через PSCI |
| Устаревшая (отстаёт от main апстрима) | Низкая | Регулярный ребейз синхронизации |
| Многословные комментарии в стиле LLM | Низкая | Очистить во время аудита |
