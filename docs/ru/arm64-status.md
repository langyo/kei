# Статус поддержки ARM64

## Поддержка ARM64

Поддержка ARM64 была внесена в проект Asterinas и независимо поддерживается в KEI.

### Текущие возможности

**OSTD (`ostd/src/arch/aarch64/`):**
- `boot/` — Точка входа BSP, загрузочные таблицы страниц
- `mm/` — Таблицы страниц ARM64, настройка MMU
- `task/` — Переключение контекста, FPU/SIMD
- `irq/` — Контроллер GICv3
- `timer/` — ARM Generic Timer (EL1)
- `trap/` — Обработка исключений EL1
- `cpu/` — Функции CPU, SMP через PSCI
- `device/` — Обнаружение устройств через FDT
- `power.rs` — Управление питанием PSCI
- `serial.rs` — Консоль PL011 UART

**Ядро (`kernel/src/arch/aarch64/`):**
- Процессы/потоки, системные вызовы, TLS, PCI, VirtIO

**OSDK:**
- Формат ARM64 `Image`, схема QEMU Arm64, CI GitHub Actions

## Стратегия KEI

Код ARM64 поддерживается непосредственно в репозитории KEI.

## Известные проблемы

| Проблема | Серьёзность | Действие |
|----------|------------|----------|
| Необходим аудит и усиление | Высокая | Аудит M2 |
| Сторонний крейт GICv3 | Средняя | Замена на встроенный драйвер |
| Только тесты QEMU | Высокая | Загрузка на NanoPi R3S |
| Нет SMP/multi-core | Средняя | Добавление PSCI secondary boot |

## Матрица тестов QEMU

| Машина QEMU | CPU | RAM | Загрузка | Примечания |
|-------------|-----|-----|----------|-----------|
| virt | cortex-a55 | 2GB | ✅ | Основная цель |
| virt | cortex-a72 | 2GB | 🔲 | Проверка на разных ядрах |
| virt | max | 4GB | 🔲 | Все функции ARM |
| sbsa-ref | max | 4GB | 🔲 | Серверный стиль загрузки |
