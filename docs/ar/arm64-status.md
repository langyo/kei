# حالة دعم ARM64

## دعم ARM64

تمت المساهمة بدعم ARM64 في مشروع Asterinas ويتم صيانته بشكل مستقل في KEI.

### القدرات الحالية

**OSTD (`ostd/src/arch/aarch64/`):**
- `boot/` — نقطة دخول BSP، جداول صفحات الإقلاع
- `mm/` — جداول صفحات ARM64، إعداد MMU
- `task/` — تبديل السياق، FPU/SIMD
- `irq/` — متحكم GICv3
- `timer/` — مؤقت ARM العام (EL1)
- `trap/` — معالجة استثناءات EL1
- `cpu/` — ميزات CPU، SMP عبر PSCI
- `device/` — اكتشاف الأجهزة عبر FDT
- `power.rs` — إدارة طاقة PSCI
- `serial.rs` — منفذ PL011 UART

**النواة (`kernel/src/arch/aarch64/`):**
- العمليات/الخيوط، استدعاءات النظام، TLS، PCI، VirtIO

**OSDK:**
- تنسيق ARM64 `Image`، مخطط QEMU Arm64، CI GitHub Actions

## استراتيجية KEI

تتم صيانة كود ARM64 مباشرة في مستودع KEI.

## المشكلات المعروفة

| المشكلة | الخطورة | الإجراء |
|---------|---------|---------|
| التدقيق والتعزيز مطلوبان | عالية | تدقيق M2 |
| crate طرف ثالث لـ GICv3 | متوسطة | استبدال ببرنامج تشغيل مدمج |
| اختبارات QEMU فقط | عالية | إقلاع على NanoPi R3S |
| لا SMP/متعدد النوى | متوسطة | إضافة إقلاع PSCI ثانوي |
